\section{Abstract}
InfoBridge is a Streamlit-based Retrieval-Augmented Generation (RAG) chatbot for Indian government services. It retrieves information from official PDF documents, builds context with source citations, and generates grounded answers through a Groq-backed LLM client. The interface includes English/Hindi UI support, a collapsible sidebar, theme persistence, and service filtering.

\section{Introduction}
Government service information is often distributed across long PDF manuals and circulars. InfoBridge addresses this by allowing users to ask service-related questions in a conversational interface and receive answers that are grounded in the indexed documents.

\section{Objectives}
\begin{itemize}
    \item Provide a searchable chatbot for government service information.
    \item Use a RAG pipeline to keep answers grounded in official PDFs.
    \item Show source citations for every assistant response.
    \item Support English and Hindi UI labels.
    \item Provide a clean Streamlit experience with theme switching and a collapsible sidebar.
\end{itemize}

\section{System Overview}
The current system follows the pipeline below:
\begin{enumerate}
    \item PDFs are extracted from the service folders under \texttt{data/}.
    \item Documents are chunked into overlapping text segments.
    \item Chunks are embedded with \texttt{sentence-transformers/all-MiniLM-L6-v2}.
    \item Embeddings and metadata are stored in FAISS.
    \item A user query is embedded and searched against the vector store.
    \item Retrieved chunks are formatted into context and citations.
    \item The Groq client generates the final answer.
    \item The UI renders the answer and source list in Streamlit.
\end{enumerate}

\section{Functional Requirements}
\begin{itemize}
    \item Accept queries through a Streamlit chat interface.
    \item Retrieve relevant document chunks using semantic search.
    \item Generate grounded answers with conversation memory.
    \item Display source citations for each answer.
    \item Filter retrieval by service category.
    \item Support English and Hindi UI labels.
    \item Persist theme selection and sidebar open/closed state within the session.
\end{itemize}

\section{Non-Functional Requirements}
\begin{itemize}
    \item Fast and responsive chat interaction.
    \item Reliable answer grounding through retrieved context.
    \item Clear readability in both light and dark modes.
    \item Maintainable modular Python codebase.
\end{itemize}

\section{Technology Stack}
\begin{itemize}
    \item Frontend: Streamlit
    \item Backend: Python
    \item Embeddings: Sentence Transformers MiniLM
    \item Vector Store: FAISS
    \item LLM: Groq chat completions
    \item PDF Parsing: PyPDF2
    \item Chunking: LangChain text splitters
\end{itemize}

\section{Current UI Behavior}
\begin{itemize}
    \item The sidebar can collapse into a left strip.
    \item The app remembers the selected theme across reruns.
    \item The language selector is English/Hindi only.
    \item The service filter uses translated labels.
    \item Assistant responses render source citations below the answer.
\end{itemize}

\section{Implementation Notes}
\begin{itemize}
    \item API key is read from the \texttt{API\_KEY} environment variable.
    \item The default model is \texttt{llama3-8b-8192} with fallback model support.
    \item The translation table is stored in \texttt{frontend/translations\_en\_hi.csv}.
    \item Session state tracks messages, memory, language, theme, and sidebar state.
\end{itemize}

\section{Verification}
\begin{itemize}
    \item Build the FAISS index using \texttt{python scripts/build\_index.py}.
    \item Run the app using \texttt{streamlit run app.py}.
    \item Validate that answers include sources and that the sidebar/theme state persists.
\end{itemize}

\section{Conclusion}
InfoBridge is now implemented as a complete modular RAG chatbot with a current Streamlit UI, Groq-based generation, persistent theme handling, a collapsible sidebar, and bilingual interface labels.
